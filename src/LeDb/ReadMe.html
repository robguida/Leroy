LeDbService Specifications

Basics

  1. LeDbService is a singleton class
  2. The code that calls this class provides it with a full server path to the database configuration file, and the data source name of the initial request.
  3. The database credentials are a json encode array stored in a json file.
  4. @todo: Additional data source names can be added as needed, but the object can handle only one configuration file, so additional dsn's must exist in the file used to instatiate the object.

Persistent Data

  1. The database configuration file is stored as an array for the entire life of the object.
  2. The PDO objects stored in an array, one PDO object per unique connection string
  3. @todo: The dsn will be set, but update-able with each request.

Output

  1. The service object does not persist the output. Rather, it returns a LeDbResultInterface object.
  2. A LeDbResultInterface object will store the PDOStatement object so that it can perform different function on the record set.
  3. The default LeDbResultInterface object is LeDbResult.

Improvements

  1. The system now stores the database configurations for one single dsn, however, it needs to store the entire contents of the json file as an array.
  2. There will be a current dsn, that can be overwritten using a setter function.
  3. Explore the possibility of linking sql statements with the dsn, so that it would not be necessary to set the dsn.
